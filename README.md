Приложение — упрощенная система управления складом (WMS) на WinForms и C# для автоматизации учета готовой продукции, инвентаризации и аналитики. Фокус на базовых процессах без аппаратных интеграций. Цель — контроль остатков и движения товаров в малом/среднем бизнесе.

Функциональные требования
Управление номенклатурой
Ведение справочника продукции: артикул, партия, серийный номер, срок годности, размер, вес. Привязка к местам хранения (ячейки, стеллажи, паллетоместа).

Учёт движения товара
Приёмка готовой продукции из производства (по накладной / без). Отгрузка (комплектация заказов, формирование упаковочных листов). Внутрискладские перемещения (смена ячейки, объединение/разделение паллет).

Инвентаризация
Создание плана-графика (полная / выборочная / циклическая). Печать списков и заданий для сотрудников (по зонам, типам товара). Автоматическое сравнение фактических остатков с учётными. Выявление расхождений (излишки, недостача, пересортица). Поддержка повторного пересчёта проблемных позиций. Формирование актов (ИНВ-3, ИНВ-19, сличительные ведомости).

Контроль и аналитика
Дашборд: остатки, резерв, сроки годности, занятость ячеек. Отчёт «старение запасов» (FIFO/FEFO). Журнал событий (кто, когда, что сделал). Отчёты по инвентаризациям (динамика расхождений). Прогнозирование необходимости пересчёта по оборачиваемости.


Нефункциональные требования
Технологии: C# WinForms, .NET Framework/Core, локальная БД (sqlite). Производительность: до 500 операций/мин, ручной ввод данных. Интерфейс: русский, таблицы с фильтрами/поиском.

Ролевая модель: кладовщик. Запрет редактирования завершённых инвентаризаций, резервное копирование, логирование.

Администрирование и безопасность
 Запрет редактирования завершённых данных. Резервное копирование. Логирование изменений.

## Реализованное решение

Приложение собрано как desktop monolith с чистым разделением слоев:

- `src/WarehouseManager.Domain` — доменная модель склада: товар, ячейка, остаток, движение, инвентаризация, аудит.
- `src/WarehouseManager.Application` — DTO и контракты прикладного слоя.
- `src/WarehouseManager.Infrastructure` — SQLite persistence на EF Core, seed-данные, сервисы складских операций.
- `src/WarehouseManager.WinForms` — русский WinForms UI для дашборда, остатков, движений, инвентаризаций и администрирования.
- `tests/WarehouseManager.Tests` — интеграционные проверки критичной бизнес-логики.

## Почему этот стек

- `WinForms + .NET 8` — прямое попадание в требования, стабильный Windows Desktop runtime и предсказуемый деплой.
- `EF Core + SQLite` — локальная БД без внешней инфраструктуры, простая инициализация и прозрачная модель данных.
- `MSTest` — достаточно для быстрых integration-style тестов без усложнения test stack.
- `Dependency Injection` через `Microsoft.Extensions.DependencyInjection` — управляемая композиция слоев и упрощение дальнейшего расширения.

## Что умеет приложение

- Дашборд по остаткам, занятости ячеек, открытым инвентаризациям и партиям с критичным сроком годности.
- Учет приемки, перемещения и отгрузки по FEFO.
- Каталог текущих остатков с поиском по SKU, товару, ячейке и партии.
- Полная и циклическая инвентаризация, фиксация факта, повторный пересчет и блокировка редактирования после завершения.
- Журнал аудита и резервное копирование SQLite-базы.
- Seed-данные для мгновенного локального старта и демонстрации сценариев.

## Запуск

```powershell
dotnet restore WarehouseManager.slnx
dotnet build src/WarehouseManager.WinForms/WarehouseManager.WinForms.csproj
dotnet run --project src/WarehouseManager.WinForms/WarehouseManager.WinForms.csproj
```

При первом запуске приложение создаст SQLite-файл в `src/WarehouseManager.WinForms/bin/Debug/net8.0-windows/Data/warehouse.db` и заполнит его демонстрационными данными.

## Проверка качества

```powershell
dotnet build src/WarehouseManager.Infrastructure/WarehouseManager.Infrastructure.csproj
dotnet build src/WarehouseManager.WinForms/WarehouseManager.WinForms.csproj
dotnet build tests/WarehouseManager.Tests/WarehouseManager.Tests.csproj
dotnet test tests/WarehouseManager.Tests/WarehouseManager.Tests.csproj --no-build
```
