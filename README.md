# Warehouse Manager

Настольное приложение для базового складского учета на `C#`, `WinForms`, `.NET 8` и `SQLite`.

Проект ориентирован на локальную работу без внешней инфраструктуры: база данных создается автоматически при первом запуске, интерфейс русскоязычный, основные операции со складом доступны из одного приложения.

## Что есть сейчас

### Справочники

- Добавление товаров:
  - артикул;
  - наименование;
  - размер / фасовка;
  - вес;
  - партия по умолчанию;
  - серийный номер;
  - срок годности.
- Просмотр списка товаров.
- Добавление ячеек хранения:
  - код ячейки;
  - зона;
  - стеллаж;
  - место.
- Просмотр списка ячеек хранения.

### Панель

- Сводка по общему остатку.
- Количество товаров в остатках.
- Количество партий с истекающим сроком годности.
- Процент занятости ячеек.
- Таблица партий с ближайшим сроком годности.

### Остатки

- Просмотр текущих складских остатков.
- Поиск по:
  - артикулу;
  - наименованию товара;
  - ячейке;
  - партии.

### Операции

- Приемка товара в ячейку.
- Перемещение товара между ячейками.
- Отгрузка товара по логике `FEFO`.
- Журнал движений по складу.

### Инвентаризация

- Создание полной инвентаризации.
- Создание циклической инвентаризации по зоне.
- Просмотр строк инвентаризации.
- Фиксация фактического количества.
- Повторный пересчет.
- Завершение инвентаризации с блокировкой редактирования.
- Формирование текстового акта расхождений внутри приложения.

### Администрирование

- Журнал аудита действий.
- Создание резервной копии базы данных.

## Что изменено по сравнению с ранней версией

- Убрано автоматическое заполнение тестовыми / демонстрационными данными.
- При первом запуске создается пустая база данных.
- Для начала работы нужно сначала создать товары и ячейки на вкладке справочников.

## Технологии

- `C#`
- `.NET 8`
- `WinForms`
- `Entity Framework Core`
- `SQLite`
- `Microsoft.Extensions.DependencyInjection`

## Структура проекта

- `src/WarehouseManager.Domain`  
  Доменная модель: товары, ячейки, остатки, движения, инвентаризация, аудит.

- `src/WarehouseManager.Application`  
  Контракты, DTO и интерфейс прикладного сервиса.

- `src/WarehouseManager.Infrastructure`  
  Работа с `SQLite`, реализация операций склада, инициализация базы, аудит, резервное копирование.

- `src/WarehouseManager.WinForms`  
  Пользовательский интерфейс.

- `tests/WarehouseManager.Tests`  
  Набор тестов проекта.

## Запуск

Из корня репозитория:

```powershell
dotnet restore WarehouseManager.slnx
dotnet build src/WarehouseManager.WinForms/WarehouseManager.WinForms.csproj
dotnet run --project src/WarehouseManager.WinForms/WarehouseManager.WinForms.csproj
```

## Где хранится база данных

После запуска файл базы создается автоматически:

```text
src/WarehouseManager.WinForms/bin/Debug/net8.0-windows/Data/warehouse.db
```

Если нужна полностью пустая база, удалите этот файл перед следующим запуском приложения.

## Рекомендуемый сценарий первого запуска

1. Открыть вкладку `Справочники`.
2. Добавить товары.
3. Добавить ячейки хранения.
4. Перейти на вкладку `Операции` и выполнить приемку.
5. Использовать вкладки `Остатки`, `Инвентаризация` и `Администрирование` для дальнейшей работы.

## Ограничения текущей версии

- Печатные формы реализованы как текстовый отчет внутри приложения, без экспорта в `PDF` или `Excel`.
- Нет отдельного UI для редактирования и удаления товаров или ячеек.
- Нет многопользовательского режима и разграничения ролей.
- Нет внешних интеграций с ERP, ТСД, сканерами и печатными устройствами.
- Интерфейс рассчитан на Windows.
